# 基本概念
## 编译型语言
需要经过编译过程才能执行。
**特点**
* 编译前检查
* 执行效率高
* 与硬件相关
* 开发周期长
**过程**
* 源代码编写
* 编译
* 链接
* 执行
## 静态类型语言
程序在编译时就能确定变量的类型,并且在编译时就能检查出大部分的类型错误。
**特点**
* 静态类型检查
* 早期错误检查
* 编译时优化
* 代码可读性强
* 执行效率高
## main入口
有且仅有一个名为main的全局函数。如果main( )返回一 个int值，则这个值将作为程序给“ 系统”的返回值。如果 main()没有返回任何值，则系统也将收到一个表示程序成功完成的值。来自main()的非零返回值表示程序执行失败。并非所有操作系统和执行环境都会用到这个返回值: 基于Linux/ Unix的环境通常会用到，而基于Windows的环境一般不会用到。

### 对象
对象是指一块连续存储区域。
#### 左值和右值
**左值**
* 表示一个有名称的存储位置，可以出现在赋值语句的左侧
* 可以取地址
* 包括变量、函数左值引用、数组元素等
**右值**
* 表示一个临时的、无名称的值，不能出现在赋值语句的左侧
* 不能取地址
* 包括字面量、表达式运算结果、函数返回值等
**纯右值**
纯粹的右值,不能被修改,比如字面量、运算表达式的结果
**将亡值**
将要被销毁的右值,比如调用返回右值引用的函数。

*左值和右值不是一个变量的属性，而是一个表达式的属性。*
```C++
int x = 5;
int y = ++x; // ++x是左值，返回被递增后的左值对象本身

int a = 5;
int b = a++; // a++是右值，表示的是递增前的值。
```
### 类型别名
```C++
using Pchar = char*;
typedef char* PChar;
```
1. 不允许在类型别名前加修饰符。
2. 当内置类型被用来表示一个可变值时，不妨给它起一个能反映其含义的别名。

## 声明
（前置修饰符）基本类型（声明符）（后缀函数修饰符）（初始化器或函数体）
**前置修饰符**
static、virtual、extern、constexpr、
**后缀函数修饰符**
const、noexcept
**声明运算符**

|     | 声明运算符      |              |
| --- | ---------- | ------------ |
| 前缀  | *          | 指针           |
| 前缀  | \*const    | 常量指针         |
| 前缀  | \*volatile | volatile指针   |
| 前缀  | &          | 左值引用         |
| 前缀  | &&         | 右值引用         |
| 前缀  | auto       | 函数（使用后置返回类型） |
| 后缀  | []         | 数组           |
| 后缀  | ()         | 函数           |
| 后缀  | ->         | 从函数返回        |
后缀声明符的绑定效果比前缀声明符更紧密
char\*kings[]是char指针的数组，而char(\*kings)[]是指向char数组的指针。
C++允许在同一条声明语句中声明多个名字，但运算符只作用于紧邻的一个名字。
### 变量
声明是一条语句，负责为程序引入一个新的名字，并指定该命名实体的类型。
声明用于定义接口。
* 类型定义一组可能的值以及一组（对象上）的操作；
* 对象是存放某类型值的内存空间
* 值是一组二进制位，具体含义由类型决定
* 变量是一个命名的对象
### 推断类型
C ++ 语言提供 了两种从表达式中推断数据类型的机制:
* auto根据对象的初始化器推断对象的数据类型，可能是变量、const或者constexpr 的 类型。
* decltype(expr)推断的对象不是 一个简单的初始化器，有可能是两数的返回类型或者 类成员的类型。
### 常量
**const**
说明接口，值在函数内不会被改变。
**constexpr**
在编译时求值，说明常量。将数据置于只读内存中。
1. **定义时机**:
    - `const`: 可以在运行时初始化,即在程序运行过程中赋值。
    - `constexpr`: 必须在编译时就能计算出结果,即编译期间就确定下来。
2. **使用场景**:
    - `const`: 可以用于任何类型,包括内置类型和用户自定义类型。
    - `constexpr`: 主要用于定义编译时常量,如数组大小、枚举值等。
3. **编译优化**:
    - `const`: 编译器可能会在某些情况下对 `const` 变量进行优化,但无法保证。
    - `constexpr`: 编译器可以对 `constexpr` 变量进行更积极的优化,如常量折叠等。
4. **内存分配**:
    - `const`: 通常存储在静态内存区域。
    - `constexpr`: 在编译期间就被计算并优化,不占用运行时内存。
5. **函数限制**:
    - `const`: 可以用于任何函数参数和返回值。
    - `constexpr`: 仅可用于纯粹的编译时常量表达式,如数学运算、类型转换等。
