OpenGL（Open Graphics Library）是一个强大的整合软硬件的跨语言、跨平台的图形API，由Khronos Group维护，用于2D和3D图形的渲染。
硬件方面，OpenGL提供了一个多级图形管线，可以使用一种名为GLSL的语言进行部分编程。
软件方面，OpenGL的API是用C语言编写的，因此API调用直接兼容C和C++。
![[Pasted image 20241005210544.png|400]]
OpenGL的高效实现一般由显示设备厂商提供，而且非常依赖于该厂商提供的硬件。
>Khronos Group是一个开放、非营利的行业协会，致力于创建可免费使用的图形合成、虚拟现实、增强现实、视觉加速、并行计算和机器学习的开放API标准，使在各种平台和设备上的应用程序和中间件都能有效利用硬件加速。
# 相关程序库
## 窗口管理库
GLFW > freeglut > GLUT
OpenGL实际上并不是把图像直接绘制到计算机屏幕上，而是将之渲染到一个帧缓冲区，然后由计算机来负责把帧缓冲区中的内容绘制到屏幕上的一个窗口中。
## 扩展库
GLEW、GL3W、GLAD
OpenGL围绕一组基本功能和扩展机制进行组织。OpenGL的核心中有一些内置的命令用来支持这一点，但是为了使用现代命令，需要执行很多相当复杂的代码，所以使用一个扩展库来处理这些细节已经成了标准做法令。
## 数学库
GLM、Eigen和vmath
3D图形编程会大量使用向量和矩阵代数。因此，配合一个支持常见数学计算任务的函数库或者类包，能极大地方便OpenGL的使用。
## 纹理图像加载库
SOIL2
使用一个纹理图像加载库，使用图像文件来向图形场景中的对象添加“纹理”。
## 模型加载器
Assimp和tinyobjloader
# 核心模式 VS 立即渲染模式
*立即渲染模式*是OpenGL的早期版本中使用的一种渲染方式。在这种模式下，开发者可以直接通过函数调用来描述图形，OpenGL会立即执行这些命令。
*核心模式*是OpenGL 3.0及以后版本的标准，旨在剔除过时的功能，鼓励使用更现代的图形编程方法，如使用缓冲区对象和着色器。

| 核心模式                                                                                                                          | 立即渲染模式                                                                                                            |
| ----------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| - **简单直观**：使用`glBegin`和`glEnd`包围的绘制命令使得图形定义简单。<br>- **性能较低**：每个绘制调用都将数据传递到GPU，效率较低，不适合复杂场景。<br>- **不支持现代OpenGL特性**：如着色器、VAO等。 | - **性能更高**：通过使用顶点缓冲区和顶点数组，减少与GPU的交互频率。<br>- **支持现代特性**：如着色器、帧缓冲对象（FBO）等，适合复杂的绘制需求。<br>- **更灵活**：设计上允许开发者更好地控制渲染过程 |
| 适合小型应用或教育目的。                                                                                                                  | 适合专业开发。                                                                                                           |

## 着色器
(1)使用C++获取GLSL着色器代码，既可以从文件中读取，也可以硬编码在字符串中。
(2)创建OpenGL着色器对象，并将GLSL着色器代码加载到着色器对象中。
(3)用OpenGL命令编译并连接着色器对象，将它们装载到GPU。
```C++
unsigned int vertexShader = glCreateShader(GL_VERTEX_SHADER);  
glShaderSource(vertexShader, 1, &vertexShaderSource, NULL);  
glCompileShader(vertexShader);  
// check for shader compile errors  
int success;  
char infoLog[512];  
glGetShaderiv(vertexShader, GL_COMPILE_STATUS, &success);  
if (!success) {  
    glGetShaderInfoLog(vertexShader, 512, NULL, infoLog);  
    std::cout << "ERROR::SHADER::VERTEX::COMPILATION_FAILED\n" << infoLog << std::endl;  
}
```
