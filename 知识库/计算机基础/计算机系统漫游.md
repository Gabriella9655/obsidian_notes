一台计算机可抽象以图灵机（Turing Machine）为计算理论模型，以冯诺依曼架构（ Von Neumann Architecture）为结构理论模型。

## 图灵机
**可计算性** ：任何算法问题，只要能分解成如图灵数学模型的读写、移动、状态转换步骤，就一定能在一台图灵机上解决。
**通用图灵机**：一台通用图灵机，通过加载不同的规则表（程序），就可以模拟任何其他图灵机的行为。

**图灵完备**
如果一个系统能模拟出图灵机的全部能力，那么它就是“图灵完备”的。这意味着，理论上它能解决任何“可计算”的问题。

## 冯·诺依曼架构
存储程序控制原理：将指令和数据以同等地位存储在同一个存储器中，并被中央处理器（CPU）顺序执行——构成了现代计算的基石。
![[IMG-20260405203912384.png|820]]
**冯诺依曼瓶颈 (Von Neumann Bottleneck)**：由于指令和数据共享同一总线 进行传输，CPU（处理速度极快）和内存（传输速度相对较慢）之间的通路就成了一个“堵点”。CPU经常要花大量时间等待指令和数据从内存中过来，这被称为指令与数据的“取指争用”。

| 原始假设     | 现代改进                   |
| -------- | ---------------------- |
| 运算器为中心   | 存储器为中心 + DMA + 多级缓存    |
| 全局同步时钟   | 异步、事件驱动、乱序执行           |
| 串行顺序执行   | 流水线、多核、SMP、MPP、机群      |
| 通用CPU做一切 | GPU、DSP、NPU、FPGA等专用加速器 |
### “宏观冯·诺依曼 + 微观哈佛”的混合架构
程序和数据都存储在同一个物理内存（DRAM）中，CPU内部的L1缓存被分成两半：L1指令缓存（I-Cache） 和 L1数据缓存（D-Cache）。

## 硬件结构

![[IMG-20260406012654015.png]]

| 组件  | 传统架构（~2011年前）    | 现代架构（主流）               | SoC（高度集成）                 |
| --- | ---------------- | ---------------------- | ------------------------- |
| CPU | 中央处理器            | 中央处理器+内存控制器+PCIe控制器    | 中央处理器+内存控制器+PCIe控制器+PCH功能 |
| 北桥  | 独立芯片，连接CPU、内存、显卡 | 功能已集成到CPU              | 已集成至SoC                   |
| 南桥  | 独立芯片，连接北桥和I/O设备  | 独立芯片（PCH），通过DMI总线连接 PU | 已集成至SoC                   |
设备管理器是一个硬件部件（常以芯片形式存在），负责与CPU沟通，并精确地控制它管理的设备完成工作。它解决了CPU无法直接与电气特性各异的外部设备通信的难题。

## Amdahl定律
阿姆达尔（Gene Amdahl）在1967年提出，用于量化当只对系统的一部分进行优化时，整个系统能获得的最大理论加速比。**系统的整体性能提升，受限于那些无法被优化的部分**。
$$
\text{Speedup}(P,S)=\frac{T}{T'}=\frac{1}{(1-P)+\frac{P}{S}}
$$
其中， $P$是可被优化的部分所占的比例，S是可被优化比例提升的倍速。

| 定律             | 关注点           | 适用场景            |
| -------------- | ------------- | --------------- |
| **阿姆达尔定律**     | 固定问题规模下的最大加速比 | 任务规模不变，只增加处理器数量 |
| **古斯塔夫森定律**    | 问题规模可随处理器数量扩展 | 大规模科学计算、数据密集型应用 |
| **Sun & Ni定律** | 考虑存储受限下的扩展    | 内存容量成为扩展瓶颈的场合   |
